# Security Policy

## Supported Versions

Only the latest release on `main` is supported.

## Reporting a Vulnerability

Please report security vulnerabilities by opening a GitHub issue marked **[SECURITY]**
or by emailing the maintainer directly.

Do not open public issues for security-sensitive bugs until a fix is available.

## Scope

- Buffer overflows or panics in parsing or protocol logic
- Memory safety issues
- Denial of service via malformed input
- Authentication or authorization bypasses (once security module is implemented)

## Response

We aim to acknowledge reports within 72 hours and provide a fix or mitigation
within 14 days for critical issues.
